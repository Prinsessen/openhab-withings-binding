# Withings Health Binding

Track your health metrics from Withings smart scales, blood pressure monitors, activity trackers, and sleep monitors — directly in openHAB.

---

## What it does

- **Body composition** — weight, fat ratio, muscle mass, bone mass, hydration
- **Cardiovascular** — heart rate, blood pressure, SpO2, VO2 max, vascular age
- **Activity** — steps, distance, calories, elevation, heart rate zones
- **Sleep** — deep/light/REM stages, sleep score, efficiency, snoring, breathing disturbances
- **Device status** — battery level, model name, last sync — per physical device

All data is retrieved from the Withings Cloud API v2. Authorization via built-in OAuth2 web UI.

---

## Supported Things

| Thing | Type | Description |
|---|---|---|
| `withings:account` | Bridge | Withings developer app — manages OAuth2 tokens |
| `withings:person` | Bridge | A Withings user account — health data channels |
| `withings:device` | Thing | A physical Withings device — battery, model, last session |

---

## ⚠️ v2.0 — Breaking Change for Upgraders

**`withings:person` is now a Bridge (was a Thing).**

If you are upgrading from v1.x:

1. Change `Thing person` → `Bridge person` in your `.things` file
2. Add a nested `Thing device` for each physical device (or discover via Inbox)
3. Update items linking device channels: `withings:person:...:device#battery` → `withings:device:...:device#battery`
4. No re-authorization needed — existing tokens remain valid

See the full migration guide in the README.

## Quick Start

### 1. Register a Withings developer application

Go to https://developer.withings.com/ — create an app, note your Client ID and Client Secret.
Set the **Redirect URI** to `https://your-openhab:8080/callback`.
Required scopes: `user.metrics,user.activity,user.sleepevents,user.info`

### 2. Configure things

```java
Bridge withings:account:home "Withings Account" [
    clientId="YOUR_CLIENT_ID",
    clientSecret="YOUR_CLIENT_SECRET",
    redirectUri="https://your-openhab.example.com/callback"
] {
    Bridge person nanna "Nanna" [
        userId=17873009,
        pollingIntervalBody=15,
        pollingIntervalActivity=30,
        pollingIntervalSleep=60
    ] {
        // Discover via Inbox — or add manually with deviceId from discovery
        Thing device body_plus "Withings Body+" [
            deviceId="030166b1ec0906a5220da271daa01a0cab245cfd"
        ]
    }
}
```

### 3. Authorize

Open `http://your-openhab:8080/withings` in a browser and click **Authorize with Withings**.
The bridge will go `ONLINE`. Note the **User ID** shown — use it for `userId` in the person bridge.

### 4. Discover physical devices

Go to **Settings → Things → (person bridge) → Scan for Things**.
Devices appear in the Inbox. Approve them — they become child things under the person bridge.

### 5. Link items

```java
// Body
Number:Mass  Withings_Weight   "Weight [%.1f kg]"     { channel="withings:person:home:nanna:body#weight" }
Number       Withings_FatRatio "Fat ratio [%.1f %%]"  { channel="withings:person:home:nanna:body#fatRatio" }
Number:Mass  Withings_Muscle   "Muscle [%.1f kg]"     { channel="withings:person:home:nanna:body#muscleMass" }

// Device (note: withings:device, not withings:person)
String       Withings_Battery  "Battery [%s]"         { channel="withings:device:home:nanna:body_plus:device#battery" }
String       Withings_Model    "Model [%s]"           { channel="withings:device:home:nanna:body_plus:device#model" }
```

## Requirements

- openHAB 4.x or 5.x
- A Withings developer account (free)
- At least one Withings device synced to the Withings app

## Download

Download the latest JAR from GitHub Releases:  
https://github.com/Prinsessen/openhab-withings-binding/releases/latest

Place it in `/usr/share/openhab/addons/` — no restart needed.
